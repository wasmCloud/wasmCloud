Run a local OTel dashboard docker

Telemetry from this component ships via OTLP gRPC to {otlp_endpoint}. The .NET Aspire dashboard is a standalone OTLP-compatible viewer that ships as a single Docker image — start it with one command:

# Aspire dashboard: UI on :18888, OTLP gRPC ingest on :18889.
docker run --rm -it \
  -p 18888:18888 \
  -p 18889:18889 \
  -e DOTNET_DASHBOARD_UNSECURED_ALLOW_ANONYMOUS=true \
  --name aspire-dashboard \
  mcr.microsoft.com/dotnet/aspire-dashboard:latest

This example's .wash/config.yaml already points OTEL_EXPORTER_OTLP_ENDPOINT at http://localhost:18889, so once the container is up the next request lands in the dashboard.

Open http://localhost:18888, then explore Traces for the handle-request tree, Structured logs for the trace-correlated INFO lines, and Metrics for http.server.request_count / http.server.response_body.size.
